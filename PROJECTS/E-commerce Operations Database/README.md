\# E-commerce Operations Database

\*\*Status:\*\* Completed Phase 1 (SQL Infrastructure)



\## Project Overview

This project simulates a backend database for an e-commerce platform. It handles user management, product inventory, and order tracking using PostgreSQL.



\## Database Schema

The database consists of three relational tables:

\* \*\*Users:\*\* Stores customer profiles (Name, Email, City).

\* \*\*Products:\*\* Manages inventory and pricing.

\* \*\*Orders:\*\* Tracks transactions using Foreign Keys to link Users and Products.



\## Key Skills Applied

\* \*\*DDL:\*\* Created tables with `SERIAL PRIMARY KEY` and `FOREIGN KEY` constraints.

\* \*\*DML:\*\* Inserted and updated data across multiple tables.

\* \*\*Analysis:\*\* Developed complex `JOIN` queries and `Subqueries` to calculate revenue.

\* \*\*Automation:\*\* Created a `VIEW` (`v\_sales\_report`) for instant business reporting.
